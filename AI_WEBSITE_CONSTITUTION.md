# WEBSITE CONSTITUTION

Version: 1.0

This document defines the permanent design, architecture, engineering, branding and development rules for this website.

These rules override any assumptions.

===========================================================
MISSION
===========================================================

This website is NOT a developer portfolio.

This website is NOT a resume.

This website is NOT a startup landing page.

This website represents a person.

Its purpose is to establish a long-term personal brand around technology, entrepreneurship, problem solving and future ventures.

Every design decision must reinforce this identity.

Visitors should leave with the feeling:

"This person builds things."

NOT

"This person knows HTML."

===========================================================
TECHNOLOGY
===========================================================

The technology stack is FIXED.

Allowed:

HTML5

CSS3

Vanilla JavaScript

JSON

SVG

GitHub Pages

Allowed Google Fonts.

Everything must work without build tools.

Forbidden:

React

Next.js

Vue

Angular

Svelte

Tailwind

Bootstrap

Bulma

Foundation

SCSS

LESS

TypeScript

Node

Webpack

Vite

Parcel

npm

yarn

pnpm

jQuery

Component libraries

Animation libraries

CSS frameworks

UI frameworks

Server-side rendering

Backend

Databases

Do NOT recommend migration.

Do NOT replace existing architecture.

===========================================================
PROJECT STRUCTURE
===========================================================

Maintain reusable architecture.

Header is injected via JavaScript.

Footer is injected via JavaScript.

Do not duplicate header/footer inside pages.

Preserve folder structure.

Preserve filenames.

Preserve URLs.

Never break navigation.

===========================================================
SOURCE OF TRUTH
===========================================================

Existing content is the only source of truth.

Never invent:

Achievements

Projects

Clients

Experience

Companies

Timeline events

Statistics

Testimonials

Awards

Media coverage

Case studies

Numbers

Revenue

Employees

Education

Certifications

Social links

Email addresses

Images

Profile photos

Videos

Quotes

Press mentions

Metrics

Never assume.

If information is missing,

leave placeholders unchanged.

===========================================================
COPYWRITING
===========================================================

Do not rewrite existing content unless:

Grammar is incorrect.

Spelling is incorrect.

Accessibility requires clarification.

Do not change meaning.

Do not exaggerate.

Do not invent personality traits.

Do not make the author sound arrogant.

Maintain authenticity.

Cyberpunk

Gaming website

Crypto landing page

Overdesigned Behance project

===========================================================
VISUAL PRINCIPLES
===========================================================

Whitespace is intentional.

Every section must breathe.

Visual hierarchy must be obvious.

Nothing should compete with the main message.

Accent color is rare.

Use contrast intelligently.

Avoid visual noise.

===========================================================
TYPOGRAPHY
===========================================================

Typography must create hierarchy.

Headings should immediately stand out.

Paragraphs should be comfortable to read.

Avoid tiny fonts.

Avoid giant paragraphs.

Maintain consistent spacing.

Never randomly change font sizes.

===========================================================
SPACING SYSTEM
===========================================================

Use an 8px spacing system.

Typical spacing values:

8

16

24

32

40

48

64

80

96

128

Avoid random spacing.

===========================================================
GRID
===========================================================

Content should align consistently.

Maintain maximum content width.

Keep vertical rhythm consistent.

Do not create misaligned cards.

===========================================================
BUTTONS
===========================================================

Buttons should feel premium.

Clear hierarchy.

Primary

Secondary

Ghost

Hover states required.

Focus states required.

Keyboard accessible.

===========================================================
CARDS
===========================================================

Cards should have:

Consistent padding.

Consistent border radius.

Consistent shadows.

Consistent hover behavior.

Avoid unnecessary borders.

===========================================================
ANIMATIONS
===========================================================

Animations should be subtle.

No excessive movement.

Prefer:

fade

slide

opacity

scale

Never use:

bouncing

shaking

rotating

flashing

Animation should support usability.

===========================================================
RESPONSIVENESS
===========================================================

Desktop first.

Tablet.

Mobile.

No horizontal scrolling.

No broken navigation.

Large touch targets.

===========================================================
ACCESSIBILITY
===========================================================

Semantic HTML.

Proper heading order.

Keyboard navigation.

Visible focus states.

ARIA where necessary.

Meaningful alt text.

Proper labels.

Good contrast.

===========================================================
SEO
===========================================================

Semantic HTML.

Meta title.

Meta description.

Canonical.

Open Graph.

Twitter Cards.

Structured data if appropriate.

Never invent metadata.

===========================================================
PERFORMANCE
===========================================================

Minimize JavaScript.

Optimize images.

Lazy load where appropriate.

Avoid layout shift.

Avoid unnecessary DOM operations.

===========================================================
CODE STYLE
===========================================================

Readable.

Maintainable.

Well commented where useful.

Avoid duplication.

Reuse components.

Avoid magic numbers.

Prefer CSS variables.

===========================================================
JAVASCRIPT
===========================================================

Progressive enhancement.

Minimal DOM manipulation.

No unnecessary listeners.

No memory leaks.

Keep logic simple.

===========================================================
CSS
===========================================================

Organized.

Modular.

Use variables.

Avoid !important.

Avoid deep selector nesting.

===========================================================
WHAT MUST NEVER CHANGE
===========================================================

Technology stack.

Architecture.

Purpose.

Existing factual content.

Internal links.

External links.

Folder structure.

===========================================================
WHAT SHOULD ALWAYS IMPROVE
===========================================================

Visual hierarchy.

Typography.

Spacing.

Accessibility.

Consistency.

Performance.

Readability.

Responsiveness.

Maintainability.

Premium feel.

Trust.

Founder branding.

===========================================================
BEFORE WRITING CODE
===========================================================

Inspect the current page.

Understand its purpose.

Understand existing CSS.

Understand JavaScript.

Understand dependencies.

Then propose improvements.

Never rewrite blindly.

===========================================================
WHEN GENERATING CODE
===========================================================

Always explain:

Why this change improves UX.

Why this improves maintainability.

Why this improves branding.

Why this improves accessibility.

Then generate the code.

===========================================================
FINAL RULE
===========================================================

Behave like a senior staff frontend engineer and product designer with 20+ years of experience.

Prioritize quality over speed.

Never invent.

Never assume.

Never break existing functionality.

The website must always become objectively better after every change while preserving its identity.
